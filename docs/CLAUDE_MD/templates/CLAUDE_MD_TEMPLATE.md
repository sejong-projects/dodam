## 경계 규칙

<!-- AI의 행동 범위를 명확히 정의하는 가장 중요한 섹션입니다.
     Always / Ask First / Never 3단계로 분류하여
     Claude Code에게 명확한 의사결정 프레임워크를 제공합니다. -->

### ✅ Always Do

<!-- 항상 지켜야 할 규칙 목록입니다. Claude Code는 이 규칙을 무조건 따릅니다. -->

- [예: 새 API 엔드포인트에 단위 테스트를 작성합니다]
- [예: 에러 처리 시 프로젝트 표준 에러 클래스(AppException)를 사용합니다]
- [예: DTO에 유효성 검사 데코레이터를 적용합니다]
- [예: 함수에 JSDoc 주석을 작성합니다]
- [예: 커밋 전 lint와 typecheck를 통과시킵니다]

### ⚠️ Ask First

<!-- 실행 전에 사용자에게 확인을 받아야 하는 작업 목록입니다. -->

- [예: 새로운 외부 패키지를 추가할 때]
- [예: 기존 DB 스키마를 변경할 때]
- [예: 공통 모듈(shared/)의 인터페이스를 수정할 때]
- [예: 기존 API의 응답 형식을 변경할 때]
- [예: 환경 변수를 추가하거나 변경할 때]

### 🚫 Never Do

<!-- 절대 하지 말아야 할 규칙 목록입니다. Claude Code는 이 규칙을 절대 위반하지 않습니다. -->

- [예: 시크릿/API 키를 코드에 하드코딩하지 않습니다]
- [예: any 타입을 사용하지 않습니다]
- [예: console.log를 프로덕션 코드에 남기지 않습니다]
- [예: 테스트 없이 비즈니스 로직을 추가하지 않습니다]
- [예: main 브랜치에 직접 push하지 않습니다]
