# CLAUDE.md 템플릿 킷 설계 문서

**작성일:** 2026-02-27
**상태:** 승인됨

---

## 개요

dodam 프로젝트에 CLAUDE.md 템플릿 킷을 추가한다. 기존 PRD 템플릿 킷과 동일한 4파일 구조를 따르며, 개인 개발자 및 소규모 팀이 Claude Code용 CLAUDE.md를 효과적으로 작성할 수 있도록 가이드와 템플릿을 제공한다.

## 대상 사용자

- 개인 개발자 및 소규모 팀
- Claude Code를 처음 사용하기 시작하는 개발자

## 접근 방식

**PRD 킷 구조 미러링 (Approach A)** — 기존 PRD 템플릿 킷의 4파일 패턴을 그대로 따른다. 일관된 구조로 사용자의 학습 부담을 줄인다.

## 파일 구조

``` markdown
docs/
├── CLAUDE_MD/
│   └── templates/
│       ├── README.md                    # 킷 개요 및 사용법
│       ├── CLAUDE_MD_WRITING_GUIDE.md   # 작성 가이드 및 모범 사례
│       ├── CLAUDE_MD_TEMPLATE.md        # 복사-붙여넣기용 템플릿
│       └── CLAUDE_MD_CHECKLIST.md       # 품질 검증 체크리스트
```

## 파일별 콘텐츠 설계

### 1. README.md (~50줄)

킷의 진입점 역할:
- 템플릿 킷 소개 및 대상 사용자
- 사용법 (템플릿 복사 → 가이드 참조 → 체크리스트 검증)
- LLM 개발 지침 가이드와의 관계 설명
- 설계 원칙 (명시적 > 암시적, 구조화된 섹션, YAGNI)

### 2. CLAUDE_MD_WRITING_GUIDE.md (~400-500줄)

섹션별 가이드:

1. **CLAUDE.md란 무엇이며 왜 중요한가** — Claude Code가 이 파일을 어떻게 사용하는지 설명
2. **핵심 원칙** — CLAUDE.md 특화 원칙:
   - 명시적으로 작성 (암시적 표현 지양)
   - 규칙 우선순위화 (Always / Ask First / Never)
   - 산문 설명보다 코드 예시
   - 간결하게 유지 — LLM 컨텍스트 한계 고려
3. **섹션별 작성 가이드** — 템플릿의 각 섹션에 대해:
   - 무엇을 포함하고 왜 포함하는지
   - 피해야 할 흔한 실수
   - 좋은 예시 vs 나쁜 예시
4. **작성 팁** — 명령형 어조, 모호성 제거, 리터럴 리딩 테스트
5. **안티패턴** — 흔한 CLAUDE.md 실수 (너무 모호, 너무 김, 모순되는 규칙, 과잉 설계)

### 3. CLAUDE_MD_TEMPLATE.md (~200-300줄)

복사-붙여넣기용 구조화된 템플릿 섹션:

1. **프로젝트 개요** — 이름, 설명, 기술 스택
2. **프로젝트 구조** — 주요 디렉토리와 역할
3. **개발 규칙** — 코드 스타일, 네이밍, 패턴
4. **명령어** — 빌드, 테스트, 린트, 배포 명령어
5. **워크플로우 규칙** — Git 규칙, PR 프로세스, 커밋 스타일
6. **경계 규칙** — Always Do / Ask First / Never Do
7. **테스트** — 테스트 패턴, 커버리지 기대치
8. **의존성 및 도구** — 주요 라이브러리, 외부 서비스

각 섹션에 포함:
- 섹션 목적 설명 주석
- 작성할 내용을 보여주는 플레이스홀더 텍스트
- 예상 형식을 보여주는 1-2줄 예시

### 4. CLAUDE_MD_CHECKLIST.md (~100-150줄)

템플릿 섹션별 검증 체크리스트:
- 각 섹션에 구체적이고 실행 가능한 규칙이 있는가?
- 규칙 간 모순이 없는가?
- 관련 코드 예시가 포함되어 있는가?
- 파일 길이가 적정한가 (~300-500줄)?
- 경계 규칙(Always/Ask/Never)이 명확히 분리되어 있는가?
- 특정 도구에 종속적인 가정이 없는가?

## 언어 및 톤

- **언어:** 한국어 (기존 dodam 문서와 일치)
- **톤:** 실용적이고 직접적 — 기존 PRD 작성 가이드와 동일한 스타일
- **어조:** 명령형 (~하세요, ~합니다)
- 코드 블록을 적극 활용하여 예시 제공

## 기존 문서와의 관계

- README에서 `docs/GUIDE/LLM_DEV_INSTRUCTION_GUIDE.md`를 상호 참조
- 작성 가이드는 LLM 가이드보다 CLAUDE.md에 특화된 심층 내용 제공 (중복이 아닌 확장)
- 기존 파일 수정 불필요 — 순수 추가 작업

## 범위

포함:
- Claude Code용 CLAUDE.md 모범 사례 및 템플릿
- 프로젝트 수준 AI 지침에 적용되는 일반 원칙

제외:

- 자동 검증 도구
